Use Case Name
ServiceNotAvailable
Participating Actors
Started by Citizen
Flow of events
1. System displays a message on the terminal screen indicating that the request cannot be made.
2. Citizen reads the notification message. 3. System resubmits input form.
Input condition
This use case extends use cases, and handles exceptional cases of failure due to technical problems.
Output condition
The input form is resubmitted.
Quality requirements Not expected
